使用 Go 实现的完整 Trojan 代理，兼容原版 Trojan 协议与配置文件格式

TLS 隧道传输

流量特征与 HTTPS 完全一致，基于成熟的 TLS 加密体系，对抗 GFW 被动/主动检测

⚡

多路复用

基于 smux 的连接复用，减少 TLS 握手开销，降低延迟、提升并发性能

🌐

WebSocket + CDN 中转

支持 CDN 流量中转和对抗中间人攻击，同时兼容非 WebSocket 客户端

🛡️

AEAD 二次加密

Shadowsocks AEAD 加密层，防止不可信 CDN 识别和审查流量

🔀

路由分流

内建 GeoIP/GeoSite 路由模块，支持国内直连、海外代理、广告屏蔽

📡

REST API + 监控

HTTP REST API 动态管理用户，健康检查端点，Prometheus 指标输出（v0.11.0 新增）

🔐

TLS 安全加固

证书有效期检测与告警、最低 TLS 版本控制、SNI 校验（v0.11.0 新增）

📦

跨平台零依赖

单二进制文件，支持 Linux / macOS / Windows / MIPS 路由器，Docker 镜像一键部署

快速开始 ​

服务端 ​

shell
sudo ./trojan-go-next -server -remote 127.0.0.1:80 -local 0.0.0.0:443 \
    -key ./your_key.key -cert ./your_cert.crt -password your_password

客户端 ​

shell
./trojan-go-next -client -remote example.com:443 -local 127.0.0.1:1080 \
    -password your_password

Docker 部署 ​

shell
docker run --name trojan-go-next -d \
    -v /etc/trojan-go-next/:/etc/trojan-go-next \
    --network host \
    ghcr.io/corevx/trojan-go-next-next

兼容的客户端 ​

Trojan-Go-Next 服务端兼容所有支持标准 Trojan 协议的客户端：

  • v2rayN — Windows / macOS / Linux
  • Clash Verge Rev — 跨平台
  • NekoBox for Android — Android
  • ShadowRocket — iOS
  • sing-box — 通用代理平台

注意： 以上客户端支持标准 Trojan 协议。Trojan-Go-Next 扩展特性（WebSocket、多路复用、AEAD）需要直接运行 trojan-go-next 二进制文件。
