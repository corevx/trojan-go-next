TLS Tunnel Transport

Traffic identical to HTTPS. Built on mature TLS encryption to resist passive/active detection.

Multiplexing

Connection multiplexing via smux reduces TLS handshake overhead and lowers latency.

🌐

WebSocket + CDN Relay

CDN traffic relay via WebSocket over TLS, with backward compatibility for non-WebSocket clients.

🛡️

AEAD Encryption

Shadowsocks AEAD secondary encryption layer to prevent CDN traffic inspection.

🔀

Routing & Splitting

Built-in GeoIP/GeoSite routing module with proxy, bypass, and block policies.

📡

REST API & Monitoring

HTTP REST API for user management, health checks, and Prometheus metrics output (v0.11.0).

🔐

TLS Hardening

Certificate expiry detection, minimum TLS version control, and SNI verification (v0.11.0).

📦

Cross-Platform, Zero Dependencies

Single binary for Linux / macOS / Windows / MIPS routers. Docker image available.

Quick Start ​

Server ​

shell
sudo ./trojan-go-next -server -remote 127.0.0.1:80 -local 0.0.0.0:443 \
    -key ./your_key.key -cert ./your_cert.crt -password your_password

Client ​

shell
./trojan-go-next -client -remote example.com:443 -local 127.0.0.1:1080 \
    -password your_password

Docker ​

shell
docker run --name trojan-go-next -d \
    -v /etc/trojan-go-next/:/etc/trojan-go-next \
    --network host \
    ghcr.io/corevx/trojan-go-next-next

Compatible Clients ​

Trojan-Go-Next server is compatible with all clients that support the standard Trojan protocol:

  • v2rayN — Windows / macOS / Linux
  • Clash Verge Rev — Cross-platform
  • NekoBox for Android — Android
  • ShadowRocket — iOS
  • sing-box — Universal proxy platform

Note: The clients above support the standard Trojan protocol. Trojan-Go-Next extensions (WebSocket, multiplexing, AEAD) require running the trojan-go-next binary directly.
